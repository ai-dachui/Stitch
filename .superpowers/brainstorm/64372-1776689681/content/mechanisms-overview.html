各项目核心机制全景

21 个项目提炼为 5 个关键维度，点击你感兴趣的

1. 知识怎么进来（Ingest）

A

一步到位

读源 → 讨论要点 → 写摘要 → 更新相关页面
简单直接，质量靠 LLM 单次发挥

B

两步链式思考

Step 1：分析（提取实体/概念/矛盾）→ JSON
Step 2：基于分析写页面
先列提纲再写文章，质量更高

C

四阶段管线

inbox → raw → knowledge → wiki
自动分类 + 打分 + 判断时效性
最完整但也最重

2. 知识怎么连接（Cross-linking）

A

LLM 自由发挥

写页面时顺手加 [[wikilinks]]
简单但容易漏

B

复合评分系统

6 个信号打分，≥6 高置信度加链接
系统化，不漏不乱

C

知识图谱 + 社区检测

四信号关联 + Louvain 聚类
自动发现知识群落

3. 知识怎么保持健康（Lint）

A

基础检查

破损链接、孤立页面、矛盾、陈旧声明、索引一致性、数据缺口
够用，每 10 次 ingest 跑一次

B

高级检查 + 来源漂移

加：推测过多警告、碎片化标签检测、PII 检测
还能建议"下一步该研究什么"
更完善，适合长期运营

4. 知识怎么活起来（记忆代谢）

A

不需要，静态编译就够

知识存进去、连起来、定期体检就行
简单可靠，适合起步

B

晶体涌现

从多条记忆中自动提炼高层洞察
"我在三个项目里都遇到了同一种问题"
让知识产生"化学反应"

C

发散扫描

找没有直接链接但共享邻居的知识对
发现跨领域的深层联系
最有创造力，但需要 200+ 页面

D

完整代谢（衰减+涌现+发散）

常用的变强、不用的变弱、碰撞产生新认知
最强大但最复杂，500+ 页面才有意义

5. 经验怎么积累（复利工程）

A

要 — 解决问题自动文档化

检测到问题解决 → 自动提取经验 → 下次遇到直接调用
第一次研究，之后只需几分钟

B

暂时不需要

先把知识库本身搞好再说

多选你感兴趣的机制，或者直接在终端告诉我你的想法。这些机制可以自由组合。
