各项目核心机制全景

21 个项目提炼为 5 个关键维度，点击展开详情

1. 知识怎么进来（Ingest）

A

一步到位（second-brain）

读源 → 讨论要点 → 写摘要页 → 更新实体/概念页 → 更新 index + log

简单直接，质量靠 LLM 单次发挥

B

两步链式思考（llm-wiki-skill / nashsu）

Step 1 分析：提取实体、概念、论点、矛盾 → 输出结构化 JSON
Step 2 生成：基于分析结果写/更新页面

先列提纲再写文章，质量显著更高

C

四阶段管线（pagefly）

inbox → raw → knowledge → wiki
自动分类器打分（relevance 1-10）+ 判断时效性 + 提取核心主张

最完整但也最重

2. 知识怎么连接（Cross-linking）

A

LLM 自由发挥（大多数项目）

写页面时顺手加 [[wikilinks]]，简单但容易漏

B

复合评分系统（obsidian-wiki）

6 个信号打分：精确匹配+4、共享标签+2、跨类别+2、外围→枢纽+2...
≥6 分高置信度加链接，3-5 中等，1-2 跳过

C

知识图谱 + 社区检测（nashsu / ctx）

四信号关联模型 + Louvain 社区检测，自动发现知识聚类

3. 知识怎么保持健康（Lint）

A

基础 8 项检查（second-brain）

破损链接、孤立页面、矛盾、陈旧声明、缺失页面、缺失交叉引用、索引一致性、数据缺口

B

高级 9 项 + 来源漂移（obsidian-wiki）

在基础上加：来源标记漂移（推测太多警告）、碎片化标签检测、PII 检测
还能建议"下一步该研究什么"

4. 知识怎么活起来（TideMind 记忆代谢）

A

突触缩放 — 自然遗忘

全局冷却，高连接节点衰减慢（自然保护重要知识）
不用的链接逐渐变弱，强度 < 0.05 自动删除

B

晶体涌现 — 自动提炼洞察

从多条具体记忆中提炼更抽象的认知
例："我在三个不同项目里都遇到了同一种架构问题"

C

发散扫描 — 跨领域连接

找两条没有直接链接但共享多个共同邻居的记忆
判断是否有跨领域的深层联系 — 这就是"化学反应"

5. 经验怎么积累（compound-engineering）

A

复利工程 — 解决问题自动文档化

检测到问题解决 → 4 个子代理并行（分析、提取、查重、历史）→ 5 维重叠评估 → 写入或更新文档
第一次解决需要研究，文档化后下次只需几分钟

你可以多选你感兴趣的机制，或者直接在终端告诉我你的想法。
这些机制不是非此即彼的，可以自由组合。
