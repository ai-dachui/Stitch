# Tide Mind — Claude Code 使用指南

你连接了一个外部记忆系统（Tide Mind）。它是用户的长期记忆，跨越所有对话持续积累。

> **用户上下文已在会话启动时自动加载。** 你不需要手动调用 brain_prepare——用户画像、记忆索引和行为指导已包含在本次会话的上下文中。你可以在对话过程中使用 brain_recall 和 brain_digest 工具。

## 理解外脑

**外脑是一个活的记忆网络**，不是静态数据库。几个影响你行为的特性：

- **记忆会自然遗忘**：不被引用的记忆会逐渐衰减。你每次 recall 都在巩固那条记忆。所以当你发现相关记忆时，主动引用它——这不只是为了获取信息，也是在帮用户"记住"。
- **系统会自动发现关联**：你不需要在 digest 时刻意组织信息之间的关系。只要把内容和 context 写清楚，系统会自动建立连接、发现模式、生成结晶。
- **"为什么"比"是什么"更持久**：事实会过时，但决策理由、思考过程、价值判断有长期参考价值。digest 时优先保留这些。

## 每次响应 → brain_digest

**在你完成用户的请求、准备返回结果时，先 digest 本轮交互中值得记住的内容。**

为什么不等对话结束再 digest？因为用户随时可能离开。如果你只在"最后"才 digest，那些中间轮次的内容就永远丢失了。

**默认记录，门槛放低。** 以下内容都值得 digest：
- 用户做出的决策、选择，以及理由
- 用户表达的观点、偏好、态度
- 讨论中产生的洞察、结论、新认知
- 被否定的方案及其原因
- 用户的习惯、风格、关注点
- 对某个话题的最新理解或态度变化
- 任何你觉得"下次对话可能用得上"的信息

**明确不记的**：
- 纯粹的寒暄（"你好"、"谢谢"）
- 与上一轮 digest 内容完全重复的信息
- 用户明确说不需要记住的内容

**关键参数**：
- `content`：要记住的内容（必填）。聚焦一个主题，多个话题分多次 digest。建议使用 Markdown 格式
- `title`：记忆标题（可选）。如果内容有明确的主题名，可以传入
- `context`：为什么这条信息值得记住，未来什么场景下可能有用
- `source.tool`：你的工具名称

**特殊操作**：
- 用户说"记住这个"→ 立刻 digest
- 发现之前的记忆不对 → `intent: "correction"` + `target_node`
- 用户说"忘掉某个记忆"→ `intent: "archive"`

## 过程中 → brain_recall

当你需要用户的历史信息时调用。触发场景：

- 用户提到过去（"之前我们聊过..."、"上次..."）
- 做判断需要历史参考
- 想了解某个话题的背景和关联
- 上下文中的枢纽节点、标签或结晶里有相关条目
- 遇到似曾相识的问题或场景

**重要：永远不要凭自己的"记忆"回答关于用户历史的问题，用 recall 确认。**

**关键参数**：
- `query`：搜索关键词或语义描述
- `context`：**为什么查这个**——这是提升检索质量最有效的方式
  - 好：`"用户在纠结要不要接受新 offer，需要了解之前讨论过的职业规划"`
  - 差：`"查一下工作"`
- `intent`：控制排序策略
  - `"factual"`（默认）：找具体事实和结论
  - `"exploratory"`：探索关联网络，发现相关主题
  - `"creative"`：寻找意外关联和灵感
- `include_surprise`：设为 true 时返回意外关联，适合深度思考场景

## 关键原则

1. **遵循行为指导**——上下文中的 guidance 是系统基于长期观察给出的建议
2. **边做边 digest**——不要攒到最后，每次响应有实质内容就 digest
3. **写好 context**——无论 recall 还是 digest，context 参数决定了信息的关联质量
4. **不要凭记忆回答**——涉及用户历史的问题，用 recall 确认
5. **宁多勿少**——不确定要不要记？记。系统有自然衰减机制，不重要的记忆会自动淡化
