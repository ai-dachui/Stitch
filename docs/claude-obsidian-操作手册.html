我需要做什么？什么是自动的？

claude-obsidian 使用者视角 — 最小操作指南

🧠 心智模型

你只需要做三件事

  • 喂 — 把素材丢进 .raw/ 或直接给 URL
  • 问 — 用自然语言提问题
  • 偶尔维护 — 隔一段时间 lint 一下

其他所有事情（提取实体、建立链接、更新索引、管理上下文、git 提交）都是自动的。

🟢 完全自动（你不需要做任何事）

🟢 自动
加载上下文
SessionStart Hook
每次打开 Claude Code session，自动读取 wiki/hot.md 注入上下文。Claude 立即知道你最近在研究什么。
每次启动 session 时自动触发，无需任何操作。
🟢 自动
上下文恢复
PostCompact Hook
长会话中 Claude Code 压缩上下文后，自动重新加载 hot.md。不会因为对话太长而丢失 wiki 状态。
Claude Code 自动 compact 时触发。你感知不到。
🟢 自动
Git 自动提交
PostToolUse Hook
每次 Claude 写入或编辑文件后，自动 git add wiki/ .raw/ 并提交。所有变更可回溯。
每次文件操作后自动触发。你永远不需要手动 git commit wiki。
🟢 自动
更新 Hot Cache
Stop Hook
Claude 每次回复结束时，自动把本次对话的关键变更写入 hot.md。下次 session 能续上。
每次 Claude 回复后自动触发。
🟢 自动
去重检查
ingest 时内部逻辑
ingest 时自动计算文件 MD5 对比 .manifest.json。相同文件不会重复编译（除非你说 "force ingest"）。
每次 ingest 的第一步，自动执行。
🟢 自动
交叉引用
ingest 时内部逻辑
ingest 时 Claude 自动在所有新页面中添加 [[wikilinks]]，链接到已有的实体和概念页面。
ingest 过程中自动完成，不需要你指定"链接到哪"。
🟢 自动
矛盾标注
ingest 时内部逻辑
新素材与已有知识冲突时，自动在两个相关页面添加 [!contradiction] callout 并互相链接。
ingest 过程中如果检测到冲突，自动标注。
🟢 自动
更新 index/log
ingest 时内部逻辑
每次 ingest 自动更新 index.md（加新页面条目）和 log.md（追加操作记录）。
ingest 的最后几步，自动执行。

🔵 需要你手动触发

🔵 手动
Ingest 素材
你说一句话
告诉 Claude 要编译什么。可以是文件名、URL、或"把这些都 ingest"。
频率：每次发现好内容时
ingest 文件名.md
ingest https://good-article.com
ingest all of these
🔵 手动
查询知识
你提问
用自然语言问任何问题，Claude 从 wiki 里综合回答（不是从训练数据）。
频率：随时需要就问
what do you know about X?
query deep: 全面对比 A 和 B
🔵 手动
保存对话精华
你觉得值得留
当一次对话产生了有价值的洞察，一键存入 wiki 永久保留。
频率：觉得"这个分析不错"时
/save 或 /save 关于XX的分析
🔵 手动
健康检查
定期维护
让 Claude 检查 wiki 的健康状态：孤页、死链、缺失引用、矛盾等。生成修复建议。
频率：每 10-15 次 ingest 后，或每周一次
lint the wiki
🔵 手动
深度研究
你想深挖某个主题
Claude 自动执行多轮搜索→抓取→综合，结果直接写入 wiki。
频率：想深入一个新领域时
/autoresearch Agent 记忆系统设计
🔵 手动
放文件到 .raw/
你决定什么值得收
把值得编译的文件（文章、笔记、转录）放到 .raw/articles/ 目录。这一步是"人类策展"。
频率：收集到好素材时，直接复制文件到目录。然后 ingest。

🟡 半自动（你触发，但过程是自动的）

🟡 半自动
Lint 后的修复
你确认后 Claude 执行
lint 发现问题后会问你"要不要自动修复？" 安全项（补 frontmatter、加链接）自动修，危险项（删页面、合并）需你同意。
lint 之后，Claude 会展示结果并询问。
🟡 半自动
Ingest 时的讨论
可选交互
ingest 一篇文章时 Claude 可能问"要强调什么？颗粒度多细？"。你可以回答细化方向，也可以说"just ingest it"跳过。
单篇 ingest 时可能出现（批量模式自动跳过）。

🚫 你永远不需要做的事

手动写 frontmatter
Claude 自动为每个页面生成完整的 YAML frontmatter（type/title/tags/status/related/sources）。
手动加 [[wikilinks]]
Claude 在 ingest 时自动识别并链接所有已知实体和概念。
手动更新 index.md
每次 ingest 自动更新索引。你不需要维护目录。
手动 git commit
PostToolUse hook 自动提交所有 wiki 变更。
手动整理笔记结构
Claude 自动决定页面放在 sources/ entities/ concepts/ 哪个目录。
手动处理"上次聊到哪了"
Hot Cache + SessionStart hook 自动恢复上下文。

💡 使用心态

把它当成一个认真做笔记的助手，不是一个聊天机器人。

  • 你负责策展（决定什么值得收进来）
  • AI 负责编译（提取、结构化、互联、标注）
  • 你负责提问（利用积累的知识）
  • AI 负责维护（去重、检查健康、保持一致性）

知识库的价值 = 你喂了多少好东西 × 你问了多少好问题。两者缺一不可。
