## 一、最终技术选型

### 主方案：claude-obsidian（v1.9.2）

| 维度 | 选择 | 理由 |
|------|------|------|
| **基座框架** | AgriciDaniel/claude-obsidian | MIT协议 · Claude Code原生 · 15 Skill模块化 · Hot Cache解决失忆 |
| **Vault 位置** | D:\code\personal-context | 就地改造，不搬文件 |
| **分支** | feat/wiki-layer | master 保持干净 |
| **检索层** | 暂用原生（hot→index→drill），规模大了接 qmd v3 | 当前 289 页面不需要向量搜索 |
| **可视化** | Obsidian Graph View + 自制 HTML 力导向图 | 原生够用 |

### 不选的方案及原因

| 项目 | 不选原因 |
|------|----------|
| nashsu/llm_wiki 桌面应用 | GPL-3.0传染性；核心能力锁在桌面应用里；skill版只是只读接口 |
| nashsu/llm_wiki_skill | 4个md文件，只能查询不能写入，依赖桌面应用跑着 |
| garrytan/gbrain | 绑定OpenClaw生态；需要Postgres；架构好但不适合Claude Code用户 |
| 方案D（双系统并行） | ROI不高，两套schema维护成本翻倍，claude-obsidian自身已够用 |

## 三、系统架构

```
personal-context/ (分支: feat/wiki-layer)
│
├── .raw/articles/              ← 素材输入层（194篇，7子目录）
│   ├── 01-ai-thinking/
│   ├── 02-ai-tools/
│   ├── 03-content-method/
│   ├── 04-business/
│   ├── 05-final-articles/
│   ├── 06-inbox/
│   └── 07-scene-blocks/
│
├── wiki/                       ← LLM 编译层（289页面）
│   ├── index.md                知识总目录
│   ├── hot.md                  近期上下文缓存（~500词）
│   ├── log.md                  操作日志（只追加）
│   ├── overview.md             全局概览
│   ├── sources/                183 个来源摘要
│   ├── entities/               39 个实体（人物/组织/产品）
│   ├── concepts/               67 个概念（理论/方法/框架）
│   ├── comparisons/            对比分析（待填充）
│   ├── questions/              问答存档（待填充）
│   └── meta/
│       ├── graph-data.json     图谱数据
│       └── knowledge-map.html  交互式知识地图
│
├── purpose.md                  ← Wiki 灵魂（目标/问题/范围）
├── WIKI.md                     ← Schema 参考文档
│
├── zones/                      原有深度内容（不动）
├── content-pipeline/           原有定稿文章（不动）
├── inbox/                      原有待处理素材（不动）
├── opinions/                   原有观点系统（不动）
└── scene_blocks/               原有技术笔记（不动）
```

### Hook 自动化

| Hook | 触发时机 | 动作 |
|------|----------|------|
| SessionStart | 每次开 session | 自动加载 wiki/hot.md |
| PostCompact | 上下文压缩后 | 重新加载 hot.md |
| PostToolUse | 每次文件写入 | 自动 git add + commit |
| Stop | Claude 回复结束 | 更新 hot.md |

## 四、日常操作规范

### 你需要做的（手动）

| 操作 | 命令 | 频率 |
|------|------|------|
| 喂素材 | `ingest 文件.md` 或 `ingest https://url` | 每天 |
| 查知识 | `what do you know about X?` | 随时 |
| 存对话 | `/save` | 有价值时 |
| 健康检查 | `lint the wiki` | 每周或每15次ingest |
| 深度研究 | `/autoresearch 主题` | 按需 |

### 自动的（不用管）

- 加载上下文（Hot Cache）
- Git 提交
- 交叉引用（[[wikilinks]]）
- 矛盾标注（[!contradiction]）
- 更新 index/log/hot
- 去重检查（MD5 哈希）

## 五、已知局限与后续规划

### 当前局限

1. **无知识淘汰** — 页面只增不删，矛盾标注不自动清理旧信息
2. **无向量搜索** — 当前靠 hot→index→drill，>500页后需接 qmd v3
3. **无自动 ingest** — 文件放进 .raw/ 后仍需手动触发 ingest
4. **无图谱可视化深度** — 只有 Obsidian 原生 graph view，没有 Louvain 社区检测
5. **Plugin 模式无 Obsidian 配置** — Graph View 颜色分组需手动配置

| 优先级 | 事项 | 来源 | 预计工作量 |
