🧠 这是什么

claude-obsidian 是一个 Claude Code 插件，把你的 Obsidian 笔记库变成一个自组织的 AI 第二大脑。
你只管丢素材和问问题，AI 自动完成编译、归档、交叉引用、矛盾标注和持续维护。

核心原则：知识编译 vs RAG

传统 RAGLLM Wiki（本项目）
知识处理时机每次查询时临时检索+拼凑素材进入时一次性编译，持续维护
知识积累❌ 不积累，每次从零开始✅ 复利增长，每次 ingest 更新全图
交叉引用❌ 不存在✅ 自动建立 [[wikilinks]]
矛盾检测❌ 不检测✅ 自动标注 [!contradiction]
适用规模任意大（靠向量搜索）< 500 篇效果最佳
类比：RAG = 每次考试从课本临时翻答案 · LLM Wiki = 认真做了笔记，笔记越来越厚越有结构

⚡ 完整能力清单

📥wiki-ingest — 知识编译
丢入任何素材（文件/URL/图片），Claude 完整阅读后编译为 8-15 个互联 wiki 页面。自动提取实体、概念、论点。支持去重（MD5 哈希跳过）、批量、图片 OCR。
ingest 文件名.md
🔍wiki-query — 知识查询
三种深度：Quick（只读 hot.md，秒回）、Standard（3-5 页综合）、Deep（全库扫描+网络搜索）。带引用回答，不是训练数据。可选 BM25+余弦重排序检索后端。
what do you know about X?
🏥wiki-lint — 健康检查
10 类检查：孤页、死链、陈旧声明、缺失页面、缺失引用、frontmatter 空缺、空章节、陈旧索引、地址验证、语义去重。生成报告 + Canvas 地图。
lint the wiki
💾save — 对话存档
把当前对话的精华存为永久 wiki 笔记。自动判断类型（question/concept/comparison），加 frontmatter，更新 index 和 hot cache。好答案不应该消失在聊天记录里。
/save
🔬autoresearch — 自主研究
给一个主题，Claude 自动执行多轮搜索→抓取→综合→入库循环。类似 Deep Research，但结果直接写入 wiki 而非只返回文字。用 program.md 定义研究目标和约束。
/autoresearch AI Agent 记忆系统
🎨canvas — 可视化画布
在 Obsidian Canvas 上添加图片、文本卡片、PDF、wiki 页面。自动定位到 zone 区域。Lint 时自动生成 overview.canvas 知识地图。
/canvas add note [[概念名]]
🧘think — 结构化思考
10 原则思考循环：观察外部→观察内心→倾听→分析→横向连接→系统连接→直觉→接纳→创造→成长。适合架构决策、复杂问题、审计。
/think 应该用什么架构？
📋wiki-fold — 日志折叠
把 log.md 的批量条目折叠为摘要页。纯提取式（不发明信息），结构幂等，dry-run 默认。DragonScale Memory 的机制之一。
fold the log, dry-run k=3
🌐defuddle — 网页提取
用 Defuddle CLI 从网页提取干净正文，去除广告、导航、杂乱元素，减少 40-60% token。配合 ingest URL 使用。
ingest https://example.com
🔄wiki-mode — 方法论切换
一键切换笔记组织哲学：Generic（通用）、LYT（Linking Your Thinking）、PARA（Projects/Areas/Resources/Archives）、Zettelkasten（卡片盒）。
bash bin/setup-mode.sh --mode lyt
🔗wiki-retrieve — 混合检索
可选升级：BM25 稀疏搜索 + nomic-embed-text 余弦重排序。500-token 分块 + 上下文前缀（Anthropic 2024 研究）。适合大库（>100页）。
bash bin/setup-retrieve.sh
🖥️wiki-cli — Obsidian CLI 传输
通过 Obsidian CLI（v1.7+原生）直接读写笔记，无需 MCP。支持 read/write/append/search/property:set/backlinks。延迟 <50ms。
obsidian-cli write "$VAULT" "$NOTE"

Ingest 编译流程

📄 丢入素材 → 🔒 MD5 去重 → 📖 完整阅读 → ✍️ 生成页面 → 🔗 交叉引用 → ⚔️ 矛盾检测

一个源文件触达 8-15 个 wiki 页面：1 source + N entities + N concepts + index + hot + log

🪝 Hook 自动化机制

Hooks 是 claude-obsidian 的"隐形引擎"——它们在你不说话时自动工作，解决了 AI 的失忆问题。

SessionStart
自动读取 wiki/hot.md
→ 新 session 秒有上下文
🟢 每次启动
PostCompact
上下文压缩后重新加载 hot.md
→ 长会话不丢失状态
🔄 自动恢复
PostToolUse
每次文件写入后 git add + commit
→ 版本可回溯
📝 每次写入
Stop
Claude 回复结束时更新 hot.md
→ 下次 session 能续上
🛑 每次结束

Hot Cache 工作原理

Session 启动 → Hook: cat wiki/hot.md → ~500 词上下文注入 → Claude 知道你最近在干什么
解决痛点："每次新 session，AI 都是一张白纸"
Hot Cache 让 AI 在 0.1 秒内恢复最近的工作上下文。

cd D:\code\personal-context && claude

lint the wiki → 修复问题 → update hot cache → fold the log

🚀 高级功能

🌐跨项目引用
在任何项目的 CLAUDE.md 中添加 wiki 路径指引。你的运维项目、代码项目、内容项目都能读取同一个知识库。
在其他项目 CLAUDE.md 里写 wiki 路径
🐉DragonScale Memory
可选扩展：日志折叠（extractive rollup）+ 确定性地址（c-NNNNNN）+ 语义去重（cosine tiling）+ 前沿研究选题。需要 python3 + 可选 ollama。
bash bin/setup-dragonscale.sh
🔀多 Agent 并行写入
v1.7+ 支持多个 sub-agent 同时写 wiki。文件级 advisory lock 防冲突，PostToolUse hook 延迟提交直到锁释放。
bash bin/setup-multi-agent.sh
📊Dataview 仪表盘
wiki/meta/dashboard.md 用 DQL 查询生成卡片网格。Obsidian Bases 原生数据库视图。自动按 status/type/domain 分组。
安装 Dataview 插件后自动生效
⚔️矛盾标注
新旧信息冲突时，在两个页面都写入 [!contradiction] callout，包含对方链接和冲突内容。知识是演化的，矛盾是特性不是 bug。
ingest 时自动检测
📈知识生命周期
每个页面有 status 字段：seed（萌芽）→ developing（发展）→ mature（成熟）→ evergreen（常青）。Lint 会提醒长期停留在 seed 的页面。
frontmatter: status: mature
