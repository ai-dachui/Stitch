claude-obsidian 使用指南

自组织 AI 第二大脑 — 让知识像复利一样积累
v1.9.2 · MIT 协议 · 基于 Karpathy LLM Wiki 模式

核心理念

"Wiki 是产品，Chat 只是接口。知识像复利一样积累。"
📄 丢入素材 → 🧠 Claude 编译 → 📚 8-15 个互联页面

每个新素材让整个 Wiki 更聪明。跨主题的关联自动发现。知识不是存了就完，而是持续编译、交叉引用、标记矛盾。

claude-obsidian 不是一个"AI 聊天插件"。它是一个 Claude Code 插件 + Obsidian 知识库，让 AI 自动组织、维护、演化你的知识。你只需要丢素材和提问题。

传统 RAG ✕ 每次查询从零检索
LLM Wiki ✓ 知识编译一次，持续维护，复利增长

安装（2 分钟）

方式一：从 Claude Code 安装插件（推荐）
1
添加市场源
claude plugin marketplace add AgriciDaniel/claude-obsidian
2
安装插件
claude plugin install claude-obsidian@agricidaniel-claude-obsidian
3
进入你的 vault 目录，启动 Claude Code
cd ~/my-vault && claude
4
输入 /wiki — Claude 自动检测环境并引导 scaffold
方式二：克隆为全新 Vault
1
git clone https://github.com/AgriciDaniel/claude-obsidian
2
cd claude-obsidian && bash bin/setup-vault.sh
3
Obsidian → 管理 Vaults → 打开文件夹 → 选择 claude-obsidian/

知识库目录结构

my-vault/ ├── purpose.md # Wiki 的灵魂：目标、关键问题、研究范围 ├── WIKI.md # Schema 参考文档 ├── .raw/ # 素材输入区（不可变） │ ├── articles/ # 文章、网页剪藏 │ ├── transcripts/ # 视频/音频转录 │ ├── screenshots/ # 截图、图片 │ └── .manifest.json # 去重追踪（自动维护） ├── wiki/ # LLM 编译层（Claude 全权维护） │ ├── index.md # 知识总目录 │ ├── hot.md # 近期上下文缓存（每 session 自动加载） │ ├── log.md # 操作日志（只追加） │ ├── overview.md # 全局概览（自动更新） │ ├── entities/ # 人物、组织、产品、仓库 │ ├── concepts/ # 理论、方法、框架 │ ├── sources/ # 来源摘要 │ ├── comparisons/ # 对比分析 │ ├── questions/ # 问答存档 │ └── meta/ # lint 报告、dashboard └── .vault-meta/ # 运行配置 ├── transport.json # 通信方式（CLI/MCP/文件系统） └── mode.json # 方法论模式

命令速查

在 Claude Code session 中直接输入即可触发对应 Skill：

📥 内容输入

ingest 文件名.md
读取 .raw/ 中的文件，编译为 wiki 页面
→ 创建 8-15 个互联页面，更新 index/hot/log
ingest all of these
批量处理 .raw/ 中所有未 ingest 的文件
→ 逐个编译 + 最后做交叉引用扫描
ingest https://example.com/article
抓取网页 → 清理 → 保存到 .raw/ → 编译
→ 用 defuddle 去广告，保存后自动 ingest

🔍 查询知识

what do you know about X?
标准查询：hot.md → index → 3-5 个相关页面 → 综合
→ 带引用的综合回答（不是训练数据，是 wiki 内容）
query quick: X 是什么？
快速模式：只读 hot.md（~1,500 token）
→ 秒回，适合简单事实查询
query deep: 全面对比 A 和 B
深度模式：读所有相关页面 + 可选网络搜索
→ 8,000+ token 综合分析，结果存回 wiki

🔧 维护管理

lint the wiki
10 类健康检查：孤页、死链、矛盾、缺失引用…
→ 生成 wiki/meta/lint-report-日期.md
update hot cache
刷新 hot.md 近期上下文缓存
→ 下次 session 开始时自动加载最新状态

💡 高级功能

/save
把当前对话的精华存为 wiki 笔记
→ 自动归类、加 frontmatter、更新 index
/autoresearch 主题
自主研究循环：搜索 → 抓取 → 综合 → 存入 wiki
→ 多轮迭代，直到达到研究深度
/canvas
可视化画布：添加图片/文本/PDF/wiki 页面
→ 生成 Obsidian Canvas JSON 文件
/think 问题描述
10 原则结构化思考：观察→倾听→分析→连接→创造
→ 深度思考框架，适合架构决策和复杂问题

Ingest 编译流程详解

一篇文章被 ingest 后发生什么？（触达 8-15 个页面）
1
去重检查 — 计算文件 MD5，比对 .manifest.json。重复则跳过（除非 force）
2
完整阅读 — "Do not skim." 全文读取，不跳过
3
讨论（可选）— "强调什么？颗粒度？" 用户可跳过
4
创建来源摘要页 → wiki/sources/文章标题.md
5
创建/更新实体页 — 每个人物、组织、产品单独一页
6
创建/更新概念页 — 重要想法、框架、方法论
7
更新 overview.md — 如果大局有变
8
更新 index.md + hot.md + log.md
9
矛盾检查 — 如果与现有知识冲突，在两个页面都标注 [!contradiction]
10
交叉引用 — 所有提到的实体/概念添加 [[wikilinks]]
💡 每次 ingest 后 .manifest.json 自动更新，记录哪些页面被创建/更新。下次遇到相同文件自动跳过。

页面类型与 Frontmatter

每个 wiki 页面都有 YAML frontmatter，Obsidian Properties 面板可直接编辑：

# 实体页示例 (wiki/entities/Andrej Karpathy.md) --- type: entity title: "Andrej Karpathy" entity_type: person role: "AI researcher, OpenAI co-founder" created: 2026-04-07 updated: 2026-06-04 status: mature # seed → developing → mature → evergreen confidence: high # high / medium / low tags: - ai-researcher - openai related: - "[[LLM Wiki Pattern]]" - "[[Vibe Coding]]" sources: - "[[.raw/articles/karpathy-gist.md]]" ---
# 概念页示例 (wiki/concepts/LLM Wiki Pattern.md) --- type: concept title: "LLM Wiki Pattern" complexity: intermediate # basic / intermediate / advanced domain: "knowledge-management" aliases: ["知识编译", "Karpathy Wiki"] status: evergreen related: - "[[Andrej Karpathy]]" - "[[RAG]]" ---

8 种页面类型：

source entity concept domain comparison question overview meta

4 级生命周期： seed（萌芽）→ developing（发展中）→ mature（成熟）→ evergreen（常青）

Hot Cache — 解决"AI 失忆"

痛点：每次新 session，AI 都是一张白纸
SessionStart Hook → 自动读取 wiki/hot.md → ~500 词即时上下文

Hot Cache 是约 500 词的近期上下文摘要。每次 Claude Code session 启动时自动加载。
Session 结束时，Stop Hook 检测 wiki 变更并提示更新 hot.md。
效果：新 session 秒回，不需要重新 "你还记得吗？"

Lint — 10 类健康检查

输入 "lint the wiki" 后执行：
1
孤立页面 — 没有任何入站链接的页面
2
死链接 — [[wikilink]] 指向不存在的页面
3
陈旧声明 — 被更新来源矛盾的旧断言
4
缺失页面 — 多处提到但没有独立页面的概念
5
缺失交叉引用 — 正文提到实体但未加链接
6
Frontmatter 缺失 — 缺少必填字段
7
空白章节 — 标题下没有内容
8
陈旧索引 — index.md 指向已删除/重命名的页面
9
地址验证（DragonScale 可选）
10
语义去重（DragonScale 可选）— 嵌入余弦相似度检测重复
📋 输出报告到 wiki/meta/lint-report-YYYY-MM-DD.md。分为"可自动修复"和"需人工审核"两级。建议每 10-15 次 ingest 后运行一次。

跨项目使用

任何 Claude Code 项目都可以引用这个知识库。在其他项目的 CLAUDE.md 中加入：

## Wiki 知识库 Path: D:\code\personal-context 需要上下文时： 1. 先读 wiki/hot.md（近期缓存） 2. 不够则读 wiki/index.md 3. 需要细节则读具体 wiki 页面 不要用 wiki 回答通用编程问题。

这样你的运维项目、代码项目、内容项目都能共享同一个知识库。

使用技巧

⚡
先 ingest 再问问题
知识库的价值在于积累。先喂 5-10 篇核心文章，再开始提问。空库上提问等于用训练数据回答。
🔄
/save 保存好答案
一个精彩的分析对话，用 /save 存回 wiki。下次问类似问题时直接从 wiki 读取，不用重新推理。
🏷️
利用 status 生命周期
新页面是 seed，多次引用后升为 developing，验证后升为 mature。Evergreen 是永远有效的核心概念。
⚔️
矛盾不是 bug
[!contradiction] callout 是特性。新旧信息冲突时双向标注，让你后续裁决。知识是演化的，不是静态的。
📊
定期 lint
每 10-15 次 ingest 后跑一次 lint。它会告诉你哪些知识是孤岛、哪些概念该有独立页面但还没有。
🌐
URL 直接 ingest
看到好文章直接 "ingest https://..." 不用手动保存。Claude 会用 defuddle 去杂质后自动编译。
